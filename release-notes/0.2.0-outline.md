# Release Notes Outline — v0.2.0 (draft)

> Scope: recent `main` changes merged in Loops 23–30 (Python parity + strict LSP hardening + release prep).

## 1. Summary (for top of release note)

- Python support reached parity target for strict/mock and strict refs/defs routes.
- Python real-LSP E2E coverage was added behind opt-in environment gates.
- LSP strict behavior now reports clearer, context-rich capability errors.
- Benchmark output now includes per-language summaries.
- Flakiness reductions and deterministic ordering/dedup improvements landed in changed/impact paths.

## 2. Highlights

### Python parity (core)
- Added `.py` profile/path handling and Python language mapping in seed/path flows.
- Added Python query/spec support for call extraction:
  - bare calls `foo()`
  - receiver calls `obj.m()` / `self.m()`
- Extended graph/changed pipelines to include Python files.
- Added/expanded Python fixtures for changed/impact, strict-mock callers/callees/both.

### Strict LSP behavior
- Implemented refs/definition-only route for missing `callHierarchy` cases (`callees` / `both` included).
- Implemented real probing logic in `probe_update` (non-mock path).
- Improved strict-mode errors to include language, direction, required capability hints, and current capability snapshot.

### Real-LSP E2E (opt-in)
- Added Python real-LSP fixture and opt-in-gated strict E2E tests for callers/callees/both (best-effort, skip-safe).
- Added Python server auto-detect/override notes and runtime path updates.

### Stability / regression
- Fixed order/duplicate flakes via sort/dedup in changed symbols and graph edge paths.
- Ran broad regression (check/image_build/impact/bench-impact) and recorded result.

### Docs / operations
- README/README_ja updated with Python strict LSP E2E runtime conditions.
- Known limitations explicitly documented.
- Python parity completion criteria (P-END-*) documented.

## 3. User-visible changes (draft bullets)

- **Added:** Python language detection for LSP profile/path (`.py`) and didOpen `languageId=python` routing.
- **Added:** Python call query/spec support and parser/spec test coverage.
- **Added:** Python strict-mock + refs/defs route test coverage for callers/callees/both.
- **Added:** Python real-LSP opt-in E2E scaffolding and strict direction coverage.
- **Changed:** Strict LSP error messages now specify language/direction/capability requirements.
- **Changed:** Benchmark script prints language-by-language summaries for changed/impacted symbols.
- **Fixed:** Non-deterministic ordering/duplication flakes in changed/impact internals.

## 4. Compatibility / behavior notes

- Python real-LSP strict E2E remains environment-dependent and intentionally skip-safe.
- Dynamic Python call patterns outside core forms may still be outside guaranteed extraction coverage.
- Strict mode now fails with richer diagnostics when capabilities are insufficient.
