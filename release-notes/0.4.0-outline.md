# Release Notes Outline — v0.4.0 (draft)

> Scope: changes in `v0.3.0..HEAD`.

## 1) Summary (top-of-release-note draft)

- Added Python non-LSP analyzer path end-to-end (`symbols`, `unresolved refs/imports`, explicit `--lang python` dispatch) with fixture-backed CLI integration tests.
- Added Auto policy operation for `--engine auto` via `--auto-policy compat|strict-if-available` and policy-aware fallback behavior.
- Improved capability-shortage diagnostics by policy (clear strict errors + fallback reason logs in non-strict paths).
- Expanded benchmark/CI coverage with fixed heavy fixtures and policy-diff comparison mode (`TS fixed` vs `auto strict-if-available`).
- Completed release-prep regression gates (`engine_lsp`, full `cargo test`, `clippy -D warnings`).

## 4) User-visible changes (candidate CHANGELOG bullets)

### Added
- Python non-LSP analyzer support for symbols and unresolved refs/imports.
- Python fixture-based CLI integration tests for changed/impact flows.
- Auto policy option: `--auto-policy compat|strict-if-available`.
- Benchmark policy-diff mode: `--compare-auto-strict-if-available`.

### Changed
- Auto strict-if-available now prefers LSP path while preserving fallback-safe non-strict behavior.
- Strict capability errors/fallback logging now include clearer policy context.
- README/README_ja now include Auto policy operation and benchmark usage updates.

### Operations
- CI/bench/nightly workflows now provide broader strict-LSP + policy-diff coverage and artifact traces.

---

## 5) Compatibility / caution notes

- `compat` remains the default for `--engine auto` (backward-compatible baseline).
- `strict-if-available` is preference-based, not hard-fail strict; it may fallback to TS when server/capabilities/session are insufficient.
- Real-LSP paths remain environment-dependent; skip-safe/test-gated operation is preserved where appropriate.
