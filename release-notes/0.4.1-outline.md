## 3) Highlights (draft bullets)

### Precision hardening for non-LSP analyzers
- Added v0.4.1 difficult fixtures:
  - Go: generics + chained call + embedded receiver
  - Java: overload + static import + nested type
  - Python: dynamic call/import edge
- Added fixture-driven tests in language analyzers to pin behavior and prevent regressions.

### Strict-LSP performance tuning (callers/both hot path)
- Reduced redundant work in references-based traversal.
- Added localized symbol-cache reuse and duplicate caller-site filtering.
- Maintained behavior contract while improving runtime in comparable lanes.

### Artifact-driven performance validation
- Collected strict-LSP RPC/time profile snapshots for target languages.
- Produced before/after artifacts for optimization comparison.
- Added explicit stability report linking changed/impacted consistency with performance deltas.

### Nightly operations hardening
- Automated flaky classification into `install/startup/capability/timeout`.
- Added flaky-type retry policy in workflow:
  - retry for install/startup/timeout
  - no auto-retry for capability-only failures
- Added CI failure summary with cause/language/evidence/repro command.
- Documented triage/retry/escalation operational flow.

## 5) Compatibility / caution notes

- Ruby/Java strict-LSP lanes still show initialize-timeout instability in recent snapshots; marked as blocked in profile artifacts.
- Rust before/after stability report includes input-scope caveat; compare performance with caution.
- Flaky retry policy currently treats capability-only failures as non-transient (no auto-retry).
