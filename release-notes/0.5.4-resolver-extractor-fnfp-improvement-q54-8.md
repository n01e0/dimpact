## Resolver / extractor fix

Updated `src/languages/ruby_spec.rs` to improve dynamic-chain visibility for caller impact analysis:

1. **`define_method` dynamic symbol extraction in `symbols_in_file`**
   - Added extraction of method symbols declared via `define_method(:name)` / `define_method("name")`.
   - This enables dynamic methods (e.g. `emit_created`, `emit_cancelled`) to participate in call-graph linking.

2. **Alias expansion for `send/public_send` dynamic targets in `unresolved_refs`**
   - Added pre-scan of `alias_method` pairs and expansion from alias target to original target when resolving dynamic `send/public_send` calls.
   - This keeps alias-based dispatch paths visible to the resolver.
