# F53-9 Go/Java extraction precision improvement (false-positive reduction)

- Run ID: `a41dde90-e976-4d41-b878-305bffb4632b`
- Scope: reduce Go/Java extraction misclassification (誤判定) in call-reference detection

## What was fixed

### Go (`src/languages/go_spec.rs`)
- Added a non-code masking pass (`mask_go_non_code`) before call-regex extraction.
- The mask excludes matches inside:
  - line comments (`// ...`)
  - block comments (`/* ... */`)
  - interpreted strings (`"..."`)
  - raw strings (`` `...` ``)
  - rune literals (`'x'`, escaped rune forms)
- Applied masked-source scanning to both qualified and bare call extraction regexes.

### Java (`src/languages/java_spec.rs`)
- Added a non-code masking pass (`mask_java_non_code`) before call-regex extraction.
- The mask excludes matches inside:
  - line comments (`// ...`)
  - block comments (`/* ... */`)
  - string literals (`"..."`)
  - char literals (`'x'`)
  - text blocks (`"""..."""`)
- Applied masked-source scanning to qualified calls, method references, and bare calls.

## Regression tests added
- `go_unresolved_refs_ignores_comment_and_string_call_patterns`
- `java_unresolved_refs_ignores_comment_and_string_call_patterns`
